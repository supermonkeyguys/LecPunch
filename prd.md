

## 优先级说明

- **P0 (核心 MVP)：** 跑通核心业务流程的最小功能集合，没有这些 APP 就无法正常使用。
- **P1 (重点体验)：** 极大提升用户体验、防止用户产生负面情绪的关键功能。
- **P2 (进阶扩展)：** 增加产品粘性、提高管理效率的进阶功能。
- **P3 (未来规划)：** 锦上添花、游戏化的附加功能。

## P0 级：核心业务流 (必须包含)

*这是根据你提供的图片提取和补全的最基础骨架。*

### 1. 账号与团队基础 (基石)

- **用户注册/登录：** 手机号或微信一键登录。
- **团队创建与加入：** * 管理员创建团队，生成团队邀请码/二维码。
  - 成员通过输入邀请码/扫码加入团队。
- 一定要保证打卡的信息安全性，不可被伪造，不可被利用 bug 恶意刷取打卡时间
- **WIFI 绑定 (核心门槛)：** 管理员在后台录入指定团队 WIFI 的信息（SSID 或 BSSID/MAC地址）。

### 2. 核心打卡引擎 (对应图片需求)

- **WIFI 校验打卡：** 点击“上卡/下卡”时，系统强制校验当前连接的 WIFI 是否匹配团队设定的 WIFI。不匹配则拒绝上卡/下卡。
- **状态实时显示：** 首页醒目位置显示当前状态（例如：“🟢 正在打卡中 02:15:30” 或 “⚪ 未打卡”）。
- **5小时防挂机机制 (核心规则)：** 后台计时，当单次连续打卡时间达到或超过 5.0 小时，该次打卡记录自动作废，时长记为 0。

### 3. 数据与统计 (对应图片需求)

- **个人历史记录：** 列表展示每一次的上下卡时间流水。
- **多维度时长统计：** 支持切换查看 每天 / 每周 / 每月 / 每学期 的累计打卡总时长。

## P1 级：体验优化与防错 (强烈建议 V1.0 加入)

*因为有“5小时作废”的严厉规则，必须有 P1 功能来安抚用户。*

- **临界点防呆提醒：** * 打卡满 **4.5 小时**触发第一次强提醒（弹窗+系统推送）：“已连续打卡4.5小时，即将失效，请尽快下卡！”
  - 打卡满 **4.9 小时**触发最后警告。
- **团队在线状态 (“谁在团队”)：** 首页展示当前正在打卡的成员头像/列表，方便了解团队实时出勤情况。
- **断网/异常容错：** 如果用户正常下卡时 WIFI 断开，允许记录本地下卡时间，等网络恢复后自动同步至服务器（防止因网络波动导致超时）。

## P2 级：管理与激励扩展 (V2.0 迭代)

- **补卡申诉流程：** 成员因忘记下卡导致失效，可提交“补卡申请”（填写真实上下卡时间及理由），管理员通过后恢复时长。
- **请假系统：** 成员可提交请假，扣除相应的考核时长要求。
- **管理员数据导出：** 管理员端可以一键导出某段时间内的全员打卡报表（Excel格式）。
- **打卡标签 (Task Tag)：** 上卡时可选当前状态（如：开会、写代码、看文献），历史记录中可查看时间分配。
- **团队排行榜：** 按周/月展示团队内部的打卡时长 Top 榜单，营造积极氛围。
